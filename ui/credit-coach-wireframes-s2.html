Design Decisions

  • Explanation per action ("Why"): PRA FS2/23 requires explainability for AI recommendations. Each action includes reasoning.
  • Disclaimer on plan: FCA Consumer Duty — estimates must be clearly labelled as such.
  • Disputed account badge: Domain KB PD24 — customers can dispute CRA data. UI must show dispute status clearly.
  • Debt-to-income ratio: Domain KB PD3 — DBR is a key affordability metric. Shown with band classification.
  • Spending impact from FA: Product Vision Section 3.2 — augments existing FA spending insights with credit context.
  • Chart text alternative: WCAG 2.1 AA — charts must have text descriptions for screen readers.
  • Milestone modal: Design System Section 5.14 — uses modal pattern with focus trap and Escape dismiss.
